# Functional Requirements Document
## BitByBit – AI-Enhanced Gaming E-Commerce Platform

---

# Introduction

## Purpose
BitByBit is an AI-driven e-commerce platform focused on gaming products including digital games, consoles, accessories, and merchandise.

This document defines the system’s functional and non-functional requirements, architecture, and system components.

## Scope
The platform includes:
- AI-powered semantic search
- Personalized recommendations
- Dynamic pricing
- Loyalty rewards system
- Pre-order management
- Secure checkout and payments
- AI chatbot support

---

# Functional Requirements

## Core Functional Requirements

1. The system shall allow users to browse the gaming catalog using AI-powered semantic search.
2. The system shall provide personalized recommendations and AI-generated product summaries.
3. The system shall implement dynamic pricing logic.
4. The system shall store user profiles, order history, and inventory data.
5. The system shall allow users to update account information.
6. The system shall allow users to manage shopping cart quantities.
7. The system shall allow users to track order shipping status.
8. The system shall allow users to delete cart items, payment methods, and account data.
9. The system shall support secure user authentication.
10. The system shall support secure payment processing.

# Architecture Design

## Architecture Type
Three-Tier Architecture:
1. Presentation Layer
2. Business Logic Layer
3. Data Layer

---

## Presentation Layer
- React Native (JavaScript)
- Mobile-first (Android focus)
- Browser compatibility
- UI Interface
- User Input Handling

---

## Business Logic Layer
- Django (Python)
- Handles:
  - Authentication (Azure – WIP)
  - Payment routing
  - Inventory management
  - AI API communication (Groq LLM)

---

## Data Layer
- MongoDB (Hosted on Azure)
- Stores:
  - User data
  - Product data
  - Inventory
  - Order history

# Database Model

## User

| Field    | Type     | Description |
|----------|----------|------------|
| userID   | ObjectID | Primary Key |
| username | String   | Login name |
| email    | String   | Email address |
| password | String   | Encrypted password |
| address  | String   | Shipping address |
| role     | String   | Access control |

---

## Products

| Field        | Type     | Description |
|--------------|----------|------------|
| productID    | ObjectID | Primary Key |
| name         | String   | Product name |
| category     | String   | Console/Game/Merch |
| price        | Float    | Product price |
| inventoryQty | Integer  | Stock quantity |
| description  | String   | Product details |

---

## Cart

| Field     | Type     | Description |
|-----------|----------|------------|
| cartID    | ObjectID | Primary Key |
| userID    | ObjectID | Owner |
| productID | ObjectID | Product reference |
| quantity  | Integer  | Selected quantity |

---

## Orders

| Field        | Type     | Description |
|--------------|----------|------------|
| orderID      | ObjectID | Primary Key |
| userID       | ObjectID | Buyer |
| totalPrice   | Float    | Total cost |
| availability | Boolean  | In stock |
| orderDate    | DateTime | Purchase timestamp |
| status       | String   | Processing/Shipped/Delivered |
